Java引用

java
编程
发布日期:   2021-12-06
文章字数:   997
阅读时长:   4 分
阅读次数:  

Java将引用分为四种：强引用（Strong Reference）、软引用（Soft Reference）、弱引用（Weak Reference）、虚引用（Phantom Reference）。这四种强度依次逐渐减弱。它们的生命周期与垃圾回收息息相关。

一、强引用

如果一个对象具有强引用，就不会被垃圾回收器回收。即使内存不足时，JVM也不会回收它们，而是抛出OOM。

String a = "hello";  // 强引用　a引用String对象
a = null;  // 取消引用

测试配置：
JVM参数：-Xms3M -Xmx3M 初始内存为3M，最大内存为3M
测试1:

public class Ref {

  public static void main(String[] args) {
    byte[] buff = new byte[1024 * 1024 * 2]; // 申请2M内存，正常执行
  }
}

测试2:

public class Ref {

  public static void main(String[] args) {
    byte[] buff = new byte[1024 * 1024 * 3]; // 申请3M内存，OutOfMemoryError
  }
}

测试3:

public class Ref {

  public static void main(String[] args) {
    byte[] buff = new byte[1024 * 1024 * 2]; // 申请2M内存
    buff = null;  // 取消引用
    System.gc();  // 垃圾回收
    byte[] xx = new byte[1024 * 1024 * 2];  // 申请2M内存，正常执行 
  }
}

测试1、测试2表明在内存不足时引用不会被回收，测试3表明引用取消后，对象会被回收

二、软引用

如果一个对象采用了软引用，它不会被垃圾回收器回收。当内存不足时，软引用才会被回收。

// -Xms3M -Xmx3M

class SoftRef {

  public static void main(String[] args) {

    List<SoftReference<byte[]>> list = new ArrayList<>();

    for (int i = 0; i < 10; i++) {
       // i = 2, 4, 6, 8时，内存不足，回收前两个软引用 
      byte[] buff = new byte[1024 * 1024];
      SoftReference<byte[]> sr = new SoftReference<>(buff);
      list.add(sr);

    }
    
    System.gc();  // 主动gc

    for (int i = 0; i < 10; i++) {
      System.out.println(list.get(i).get());　// 除了最后两个，其他都为null
    }

  }
}

主动gc后，list中最后两个的值不为null，说明正常情况下软引用不会被回收。其他都为null说明，在内存不足的情况下，软引用会被回收。

三、弱引用

如果一个对象是弱引用时，垃圾回收器只要发现弱引用，无论内存是否充足，都会回收对象。垃圾回收线程是一个优先级较低的线程，不一定可以及时发现弱引用。

// -Xms3M -Xmx3M

class WeakRef {

  public static void main(String[] args) {

    List<WeakReference<byte[]>> list = new ArrayList<>();

    for (int i = 0; i < 10; i++) {
      byte[] buff = new byte[1024 * 1024];
      WeakReference<byte[]> wr = new WeakReference<>(buff);
      list.add(wr);

    }

    System.gc();

    for (int i = 0; i < 10; i++) {
      System.out.println(list.get(i).get());　 // 全部为null
    }

  }
}

四、虚引用

一个对象是否有虚引用的存在， 完全不会对其生存时间构成影响(虚引用并不会决定对象的生命周期，有他没他都一个样)，也无法通过虚引用来取得一个对象实例。
虚引用的的对象存活周期并不能确定，对象可能在任何时候被回收。
为一个对象设置虚引用关联的唯一目的就是能在这个对象被收集器回收时收到一个系统通知。 换句话说这就是起到了一个哨兵的作用。
使用虚引用的目的就是为了得知对象被GC的时机，所以可以利用虚引用来进行销毁前的一些操作，比如说资源释放等。这个虚引用对于对象而言完全是无感知的，有没有完全一样，但是对于虚引用的使用者而言，就像是待观
察的对象的把脉线，可以通过它来观察对象是否已经被回收，从而进行相应的处理。

五、总结

级别 被垃圾回收时间 生存时间 用途
强引用 从来不会 JVM停止运行时终止 对象的一般状态
软引用 内存不足时 内存不足是终止 对象缓存
弱引用 垃圾回收时 GC后终止 对象缓存
虚引用 随时 随时 标记、哨兵
