泛型

java
编程
发布日期:   2021-12-12
文章字数:   1.7k
阅读时长:   7 分
阅读次数:  

Java 泛型（generics）是JDK1.5中引入的一个新特性，其本质是参数化类型，解决不确定具体对象类型的问题;其所操作的数据类型被指定为一个参数（type parameter）这种参数类型可以用在类、接口和方法的创建中，分别称为泛型类、泛型接口、泛型方法。
​

一、类型擦拭

1.什么是类型擦拭？

类型擦拭是由Type Erasure翻译而来。为了兼容JDK1.5之前的版本，通过类型擦拭的方式增加泛型功能。在编译器层次上是存在泛型的，编译之后的字节码中不包含泛型中的类型信息。

public class Container<E> {
  private E value;
  
  public Container(E value) {
  	this.value = value;
  }
  
  public E getValue() {
  	return this.value;
  }
  
  public void setValue(E value) {
  	this.value = value;
  }
}

类型擦拭之后(jad反编译)：

public class Container
{

    public Container(Object obj)
    {
        value = obj;
    }

    public Object getValue()
    {
        return value;
    }

    public void setValue(Object obj)
    {
        value = obj;
    }

    private Object value;
}

编译器将类型视为Object；在适当的时候，编译器会根据实现安全的强制转换

public static void main(String[] args) {
  Container<Integer> container = new Container<>(1);
  Integer value = container.getValue();
}

擦拭之后：

public static void main(String[] args) {
  Container container = new Container(1);
  Integer value = (Integer)container.getValue();
}

代码反编译(javap -c)

在看一个Override例子

public class Container<E> {
   
  // Compile error
  // 'equals(E)' clashes with 'equals(Object)';
  // both methods have same erasure
  public boolean equals(E value) {  
  	return true;
  }
  
  // 正常
  public boolean equals(Object value) {
  	return true;
  }
}

当用equals(E value)进行重写时，会发生冲突，也可以从侧面反映在擦拭后变成Object。

2.类型擦拭的局限性

（1）不能是基本类型，例如int。类型擦拭本质上是将转化成Object类型，Object类型无法持有基本类型。不同的基本类型的内存大小是不不一样的，统一用引用类型，利于内存大小控制（也有可能是编译器作者偷懒）。
（2）无法获取带泛型的Class

Container<String> c1 = new Container<>("c");
Container<Integer> c2 = new Container<>(1);
Class clazz1 = c1.getClass();
Class clazz2 = c2.getClass();
System.out.println(clazz1 == clazz2);  // true
System.out.println(clazz1 == Container.class);  // true

System.out.println(c1 instanceof Container<String>))  // Compile error

（3）不能实例化E类型，因为擦拭后实际上是new Object()

public Class Container<E> {
	private E value;
    
    public Container() {
    	value = new E();  // Compile error
    }
}

但可以借助Class进行实例化

public class Container<E> {
  private E value;

  public Container(Class<E> clazz){
    try {
      value = clazz.newInstance();
    } catch (Exception e) {
      e.printStackTrace();
    }
  }
    
  public static void main(String[] args) {
  	Container<String> c = new Container<>(String.class);
  }
  
}

3.桥接方法

桥接方法，英文为Bridge methods。由于存在类型擦拭特性，编译后的方法可能会产生冲突。如boolean equals(Object other)和boolean equals(T other)在编译的时候不是一个东西，在编译之后又是同一个东西，从而产生冲突。
含有泛型参数的父类在被子类所继承时，指定了类型参数的的具体类型，并重写了含有参数类型的方法。在编译器进行类型擦拭之后，无法判断是否重写了方法，所以需要一个添加桥接在子类和父类之间的方法。
（1）正常方法的继承

public class NormalClass {
  private Object value;

  public void setValue(Object value) {
    System.out.println("set");
    this.value = value;
  }

  public Object getValue() {
    System.out.println("get");
    return this.value;
  }
}

class SubNormalClass extends NormalClass {

  public void setValue(Integer value) {  // 与NormalClass的setValue方法签名不一致，两个方法
    System.out.println("Sub set");
    super.setValue(value + 1);
  }

  public Integer getValue() {  // 重写，与NormalClass的getValue方法签名一致
    System.out.println("Sub get");
    return (Integer) super.getValue() + 1;
  }

  public static void main(String[] args) {
    SubNormalClass subNormalClass = new SubNormalClass();
    subNormalClass.setValue(1);
    System.out.println(subNormalClass.getValue()); 

    System.out.println("------------------------------");

    NormalClass normalClass = new SubNormalClass();
    normalClass.setValue(1);
    System.out.println(normalClass.getValue());
  }
}

结果如下：

Sub set
set
Sub get
get
3
------------------------------
set
Sub get
get
2

模拟NormalClass类是一个泛型类，将setValue和getValue的参数和返回值的类型都设置为Object。子类SubNormalClass的setValue和getValue参数和返回值的类型都设置为Integer。这相当于在没有桥接方法的情况下，存在泛型类NormalClass, SubNormalClass extends NormalClass,并且重写了setValue和getValue方法。
测试结果显示在多态的情况下，normalClass.setValue(1)实际上调用的是NormalClass的方法。无法按照预期的那样调用SubNormalClass的setValue方法。在正常类的继承中，这是正常的。如果这是在存在泛型的类中，是无法接受的，因为调用的方法可能不是预期的方法。因此，需要桥接方法。
​

（2）编译器生成桥接方法

class IntegerContainer extends Container<Integer> {

  public IntegerContainer(Integer value) {
    super(value);
  }

  @Override
  public Integer getValue() {
    return super.getValue();
  }

  @Override
  public void setValue(Integer value) {
    super.setValue(value);
  }

}

jad反编译

class IntegerContainer extends Container
{

    public IntegerContainer(Integer integer)
    {
        super(integer);
    }

    public Integer getValue()
    {
        return (Integer)super.getValue();
    }

    public void setValue(Integer integer)
    {
        super.setValue(integer);
    }

    public volatile void setValue(Object obj) // ACC_BRIDGE ACC_SYNTHETIC
    {
        setValue((Integer)obj);
    }

    public volatile Object getValue()  // ACC_BRIDGE ACC_SYNTHETIC
    {
        return getValue();
    }
}
  • ACC_BRIDGE 用来说明桥接方法是由Java编译器生成的
  • ACC_SYNCTHETIC 用来表示该类成员没有出现在源代码中, 而是由编译器生成

下面这个例子也可以从侧面看出桥接方法的存在。


class IntegerContainer extends Container<Integer> {

  public IntegerContainer(Integer value) {
    super(value);
  }

  @Override
  public Integer getValue() {
    System.out.println("yyy");
    return super.getValue();
  }

  @Override
  public void setValue(Integer value) {
    System.out.println("xx");
    super.setValue(value);
  }

  public static void main(String[] args) {
    Container<Integer> integerContainer = new IntegerContainer(1);

    System.out.println(integerContainer.getValue());  // 先打印 yyy, 再打印1
  }

}

二、泛型继承

1. 继承

public IntegerContainer extends Container<Integer> { 
}

IntegerContainer ic = new IntegerContainer(1);

父类类型是Container，子类类型为IntegerContainer。可以通过子类获取父类的泛型类型。

Class<IntegerContainer> clazz = IntegerContainer.class;
Type t = clazz.getGenericSuperclass();
if (t instanceof ParameterizedType) {
    ParameterizedType pt = (ParameterizedType) t;
    Type[] types = pt.getActualTypeArguments();
    Type firstType = types[0];
    Class<?> typeClass = (Class<?>) firstType;
    System.out.println(typeClass);  // class java.lang.Integer
}

2.参数类型继承关系与泛型继承

两个类的参数类型具有继承关系，不代表两者之间具有继承关系。

class Animal {
}

class Cat extends Animal {}

class Container<T> {}

Container<Animal> cat = new Container<Cat>();  // Compile error

三、Type类型


在没有泛型之前，所有的类型都通过Class类进行抽象，Class类的一个具体对象就代表了一个类型。
在JDK1.5增加了泛型之后，扩充了数据类型，将泛型也包含了。
JDK在原来的基础上增加了一个Type接口，它是所有类型的父接口，它的子类有

  • Class类： 原始/基本类型，包括平时我们所有的类、枚举、数组、注解，还有int、float等基本类型
  • ParameterizedType接口：参数化类型，比如List
  • TypeVariable接口：类型变量，比如List中的T就是参数化变量
  • GenericArrayType接口： 数组类型，比如List[]、T[]
  • WildcardType接口：泛型表达式类型，比如List< ? extends Number>
