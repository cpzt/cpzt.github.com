Java中Type接口

java
编程
发布日期:   2021-12-13
文章字数:   815
阅读时长:   3 分
阅读次数:  

Type体系在Java 5以后与泛型一起引入。Type放置于java.lang.reflect包，是Java语言中所有类型的公共父接口。

public interface Type {
    default String getTypeName() {
        return toString();
    }
}

Type接口包含四个子类ParameterizedType、 TypeVariable、GenericArrayType、WildcardType和一个实现类Class。

1. ParameterizedType

参数化类型，即泛型类型。如List

public interface ParameterizedType extends Type {
	
    // 获取参数类型<>里面的那些值,例如Map<K,V> 那么就得到 [K,V]的一个数组
    Type[] getActualTypeArguments();
	
    // 获取参数类型<>前面的值，例如例如Map<K,V> 那么就得到Map
    Type getRawType();
	
    // 获取其父类的类型，例如Map有一个内部类Entry,
    // 那么在Map.Entry<K,V> 上调用这个方法就可以获得Map
    Type getOwnerType();
}

2. TypeVariable

类型变量，泛型信息在编译时会被转换为一个特定的类型, 而TypeVariable就是用来反映在JVM编译该泛型前的信息。(通俗的来说，TypeVariable就是我们常用的T，K这种泛型变量)

public interface TypeVariable<D extends GenericDeclaration> extends Type, AnnotatedElement {
    // 返回此类型参数的上界列表，如果没有上界则放回Object. 
    // 如  V extends @Custom Number & Serializable 这个类型参数
    // 有两个上界，Number 和 Serializable 
    Type[] getBounds();

    // 类型参数声明时的载体
    // 如 `class TypeTest<T, V extends @Custom Number & Serializable>` 
    // 那么V的载体就是TypeTest
    D getGenericDeclaration();

    
    String getName();

    // Java 1.8加入 AnnotatedType
    // 如果这个这个泛型参数类型的上界用注解标记了，可以通过它拿到相应的注解
    AnnotatedType[] getAnnotatedBounds();
}

3. GenericArrayType

泛型数组类型，用来作为数组的泛型声明类型。例如List[] listArray， T[] tArray两个数组，其中List[], 和T[]就是GenericArrayType类型。

public interface GenericArrayType extends Type {
    // 获取泛型类型数组的声明类型，即获取数组方括号 [] 前面的部分
	Type getGenericComponentType();
}

GenericArrayType 接口只有一个方法getGenericComponentType()，其可以用来获取数组方括号 [] 前面的部分，例如T[]，在其上调用getGenericComponentType 就可以获得T. 值得注意的是多维数组得到的是最后一个[] 前面的部分，例如T[][], 得到的是T[]。
​

4. WildcardType

通配符类型，即带有?的泛型参数, 例如 List<?>中的？，List<? extends Number>里的? extends Number 和List<? super Integer>的? super Integer 。如果没有指定上边界，则默认为Object，如果没有指定下边界，则默认为String

public interface WildcardType extends Type {
    // 上边界
    Type[] getUpperBounds();

    // 下边界
    Type[] getLowerBounds();
}

在编译时，原始类型会生成字节码文件对象，而泛型类型相关的类型并不会生成与其相对应的字节码文件(因为泛型类型将会被擦除)，无法将泛型相关的新类型与class相统一。因此，为了程序的扩展性以及获取泛型的类型，引入了Type这个类型，并且新增了ParameterizedType、TypeVariable、 GenericArrayType,、WildcardType四个表示泛型相关的类型以及Class类，这样就可以用Type类型的参数来接受以上五种子类的实参或者返回值类型就是Type类型的参数。统一了与泛型有关的类型和原始类型Class。而且这样一来，我们也可以通过反射获取泛型类型参数。
